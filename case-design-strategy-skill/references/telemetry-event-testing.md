# Telemetry Event Testing Reference

## 1. 适用输入
当需求文档出现以下字段时，进入埋点测试模式：
- eventid
- 触发上报时机
- 埋点类型（页面曝光/元素曝光/点击/转化）
- 埋点参数（如 page_id、tab_flag、grade、status 等）

## 2. 事件测试最小模板
对每个事件至少生成以下测试点：
1. 正常触发上报
2. 非触发场景不上报
3. 固定参数校验
4. 动态参数校验
5. 枚举参数遍历校验
6. 重复触发/去重校验
7. 生命周期切换校验（刷新、切换、前后台、返回）

## 3. 曝光类专项
- 首次进入曝光
- 滑动进入可视区曝光
- 快速滑动/部分可见（有效曝光门槛）
- 滑出再滑入重复曝光策略
- 切换页面再返回是否重复上报

## 4. 点击类专项
- 点击热区校验（点击哪里才上报）
- 连续快速点击的防重/节流
- 不同登录态点击上报一致性
- 点击后跳转成功/失败对上报影响

## 5. 参数契约校验
- 固定值参数：值必须与定义一致（如 page_id）
- 动态参数：值来源与页面上下文一致（如 tab、grade、status）
- 枚举参数：合法值全覆盖，非法值不出现
- 参数完整性：必填参数不得缺失或空字符串

## 6. 不上报清单（必须显式设计）
至少覆盖以下“不上报”场景：
- 文档明确标注“切换/回退/恢复时不上报”的路径
- UI变化但未达触发条件
- 同一事件在去重窗口内重复触发

## 7. 证据与验收
每条埋点用例要写清：
- 查询过滤维度（设备ID/用户ID/时间窗口）
- 目标 eventid
- 期望条数或去重行为
- 关键参数断言

建议输出证据格式：
- `eventid`: xxx
- `trigger`: xxx
- `evidence`: 查询截图/日志片段/统计条数
- `assertions`: 参数与条数断言
