## Pattern A：状态矩阵与多入口冲突

### 典型优势
- 按用户路径分层，不按文档章节平铺。
- 显式覆盖状态矩阵：用户态 × 数据态 × 对象态 × 生命周期态。
- 独立建模跨页面链路：入口、跳转、返回、恢复。

### LLM 易漏边界
- 对象不满足归属条件时的“无入口展示”。
- 多入口并存时的展示优先级与冲突处理。
- 状态变更后的过滤逻辑（如取消关联、失效、撤销）。
- 多媒体/可视区相关行为（自动播放、暂停、进度恢复）。

### 可复用规则
1. 每个一级场景至少覆盖：正常 + 边界 + 异常。
2. 每条关键跳转链路都验证“返回后状态保持”。
3. 列表页必须覆盖：首屏、刷新、分页、空态、异常态。
4. 每个关键功能至少有一个“状态切换后复验”测试点。
5. 功能验证与可观测性验证（事件/日志）分层，不混写。

## Pattern B：策略链路、兜底与开关

### 典型优势
- 覆盖主策略、兜底策略、补足逻辑、循环逻辑的完整链路。
- 将策略开关单独建模，验证开/关行为差异。
- 对过滤、排序、格式化等展示规则做明确断言。

### LLM 易漏边界
- 主策略异常或数据不足时的降级与顺延补位。
- 过滤后数量不足时的补足逻辑。
- 开关开启/关闭后的结果稳定性与波动性校验。
- 缓存命中/过期路径的数据一致性与更新行为。
- 数值展示分段边界（小值/中值/大值）。

### 可复用规则
1. 推荐/决策系统成对验证：主路径 + 兜底路径。
2. 任一过滤规则后增加“数量是否补足”的测试点。
3. 开关类需求至少覆盖：开时、关时、切换后三类行为。
4. 缓存类需求至少覆盖：命中、过期、刷新后一致性。
5. 展示格式字段补充分段边界值验证。

## Pattern C：跨端一致性与口径对账

### 典型优势
- 覆盖端内行为到后台统计展示的闭环。
- 强化角色视角：不同角色的可见范围与可执行动作。
- 分层验证一致性：明细、聚合、报表展示。

### LLM 易漏边界
- 只测页面结果，不测口径定义与聚合规则。
- 只测单端，不测跨端回流。
- 只测单角色，不测权限切片差异。
- 忽略同步窗口（即时结果与延迟最终一致）。

### 可复用规则
1. 数据类需求同时验证：采集正确、聚合正确、展示正确。
2. 跨端链路至少有一条“端内操作 -> 后台可见”回流用例。
3. 权限需求按角色切片验证“可见范围 + 可执行动作”。
4. 关键指标做“明细求和 vs 汇总值”一致性校验。
5. 同步链路增加“即时 + 延迟窗口”双时点校验。

## Pattern D：时间窗口与频次约束

### 典型优势
- 覆盖触发、限流、内容模板、跳转、回收的完整链路。
- 对时间窗口、频次上限、跨对象隔离做明确约束验证。
- 覆盖终端状态差异：前台、后台、冷启动。

### 可复用规则
1. 触达类能力验证：触发条件 + 不触发条件。
2. 限流策略覆盖：未超限、临界值、超限后三种状态。
3. 时间规则覆盖：窗口内、窗口外、边界时刻。
4. 跳转链路覆盖：前台直达、后台唤醒、冷启动恢复。
5. 内容模板覆盖：短文本、长文本截断、字段拼接格式。

## Pattern E：埋点事件契约与上报时机

### 典型优势
- 将事件验证从功能用例中独立成模块，避免被功能验证淹没。
- 对每个事件建立“触发时机 + 不触发时机 + 参数契约”三元校验。
- 覆盖页面生命周期与交互切换场景（初始化、刷新、切换、返回、前后台）。

### LLM 易漏边界
- 只验证“有上报”，不验证“哪些场景必须不上报”。
- 只验证 eventid，不验证参数枚举和动态字段取值来源。
- 忽略可见性/曝光有效性规则（部分可见、快速滑动、重复曝光去重）。
- 忽略状态切换后的参数变化（登录态、tab、年级、开关状态）。
- 忽略证据闭环（日志查询条件、过滤维度、结果可追溯性）。

### 可复用规则
1. 每个事件至少覆盖：应上报场景 + 不应上报场景。
2. 每个事件至少验证：固定参数、动态参数、枚举参数三类字段。
3. 曝光类事件必须验证：有效曝光、重复曝光、去重策略。
4. 点击类事件必须验证：触发区域、快速连点、防重逻辑。
5. 每个埋点用例都要给出“如何查证”的明确路径（设备/用户过滤、eventid、时间窗口）。

## APP（C端）通用检查清单
- 登录/会话矩阵：未登录、已登录、会话失效。
- 生命周期：冷启动、热启动、前后台切换。
- 内容流：首屏、刷新、分页、空态、异常态。
- 交互链路：进入、跳转、返回、位置恢复。
- 弱网异常：超时、失败重试、降级与兜底。
- 可观测性：曝光、点击、转化事件及参数完整性。

## WEB（B端）通用检查清单
- 权限矩阵：角色 × 页面 × 按钮 × 数据。
- 查询系统：筛选、搜索、排序、分页、重置。
- 表单与批量：校验、联动、幂等、防重复提交。
- 一致性：列表与详情一致、操作前后回显一致。
- 异常治理：4xx/5xx、部分成功、回滚提示。
- 审计追溯：操作日志、变更记录、导出一致性。
